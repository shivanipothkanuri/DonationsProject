🤝 CharityAid — API Reference Base URL: http://localhost:8080
Authentication: All protected endpoints require the header Authorization: Bearer <token>. Obtain a token from POST /api/auth/login or POST /api/auth/setup.

🔑 Authentication — /api/auth
GET /api/auth/setup-needed Public

Check whether the first-time admin setup is still required. The login page calls this automatically to decide whether to show the Setup tab.

Response body: 
{ "status": "OK", "message": "OK", "data": true }

data is true when no ADMINISTRATOR account exists, false otherwise.

POST /api/auth/setup Public One-time only

Create the first Administrator account. Fails with 409 Conflict if an admin already exists.

Request body: 
{
  "fullName":    "Admin Name",
  "email":       "admin@example.com",
  "password":    "SecurePass1!",   // min 8 chars
  "phoneNumber": "+1 555 000 0000" // optional
}
Success response (201): 
{ "status": "OK", "message": "Administrator account created successfully",
  "data": { "token": "...", "userId": 1, "fullName": "Admin Name",
            "email": "admin@example.com", "role": "ADMINISTRATOR", "expiresIn": 900000 } }
POST /api/auth/register Public

Self-register as a Donor. Role is hard-coded to DONOR.

Request body: 
{
  "fullName":    "Jane Doe",
  "email":       "jane@example.com",
  "password":    "mypassword",
  "phoneNumber": "+1 555 111 2222" // optional
}
Success response (201): 
{ "status": "OK", "message": "Registration successful",
  "data": { "token": "...", "role": "DONOR", ... } }
POST /api/auth/login Public

Authenticate and obtain a JWT. Account is locked after 5 consecutive failures.

Request body: 
{
  "email":    "user@example.com",
  "password": "mypassword"
}
Success response (200): 
{ "status": "OK", "message": "Login successful",
  "data": { "token": "eyJ...", "tokenType": "Bearer", "userId": 1,
            "fullName": "Jane Doe", "email": "...", "role": "DONOR", "expiresIn": 900000 } }

Token expiry: 15 minutes (900 000 ms). Re-login to get a new token.

POST /api/auth/logout Any Auth

Log out the current user. Request is recorded in the audit trail.

POST /api/auth/forgot-password Public

Request a password reset. Sends an email with a reset token/link.

Request body: 
{ "email": "user@example.com" }
POST /api/auth/reset-password Public

Complete password reset using the token from the email.

Request body: 
{ "token": "reset_token_from_email", "newPassword": "newpass123" }
POST /api/auth/mfa/setup Any Auth

Initiate MFA (TOTP) setup. Returns a secret and QR code URI to scan with an authenticator app.

Success response (200): 
{ "status": "OK", "message": "MFA setup initiated",
  "data": { "secret": "...", "qrUri": "otpauth://totp/..." } }

Scan the QR code, then call /api/auth/mfa/confirm with the generated code.

POST /api/auth/mfa/confirm Any Auth

Confirm MFA setup by providing a code from the authenticator app.

Request body: 
{ "code": "123456" }

After confirmation, MFA is enabled on the user account.

POST /api/auth/mfa/verify Any Auth

Verify a TOTP code during login or for sensitive operations.

Request body: 
{ "code": "123456" }
DELETE /api/auth/mfa Any Auth

Disable MFA on the current user's account.

👤 Users — /api/users
GET /api/users/me Any Auth

Fetch the currently-authenticated user's profile.

Response: User object with id, fullName, email, role, status, lastLogin.
PUT /api/users/me Any Auth

Update own profile (name and phone only).

Request body: 
{ "fullName": "New Name", "phoneNumber": "+1 555 999 0000" }
PUT /api/users/me/password Any Auth

Change own password. Must provide current password for verification.

Request body: 
{ "currentPassword": "oldpass", "newPassword": "newpass123" }
GET /api/users?page=0&size=20 ADMIN

List all user accounts (paginated).

GET /api/users/{id} ADMIN

Fetch any user by their ID.

POST /api/users ADMIN

Create a Staff, Case Manager, or Leadership account. Can also create additional admins.

Request body: 
{
  "fullName":    "Staff Member",
  "email":       "staff@example.com",
  "password":    "temppass1",
  "phoneNumber": "+1 555 000 0001",  // optional
  "role":        "STAFF"             // STAFF | CASE_MANAGER | ADMINISTRATOR | LEADERSHIP
}
PATCH /api/users/{id}/role?role=STAFF ADMIN

Change a user's role. Query param role must be one of: DONOR, STAFF, CASE_MANAGER, ADMINISTRATOR, LEADERSHIP.

PATCH /api/users/{id}/deactivate ADMIN

Set user status to INACTIVE. Prevents login.

PATCH /api/users/{id}/reactivate ADMIN

Restore a deactivated or suspended user to ACTIVE status.

PATCH /api/users/{id}/suspend ADMIN

Set user status to SUSPENDED (e.g. for policy violation).

PATCH /api/users/{id}/unlock ADMIN

Unlock a LOCKED account (locked after 5 failed login attempts). Resets failed-attempt counter.

📧 Notification Templates — /api/notification-templates
GET /api/notification-templates ADMIN

List all email notification templates with their current subject and body text.

Template keys: donation_confirmation, campaign_alert, aid_approval, aid_denial, escalation, password_reset, fulfillment_notice, pending_reminder, bulk_update, high_value_donation_alert.

GET /api/notification-templates/{templateKey} ADMIN

Fetch a single template by its key. Returns the template's subject, body, and supported variables.

Example response: 
{ "status": "OK", "data": { "templateKey": "donation_confirmation",
    "subject": "Thank you for your donation!",
    "body": "Dear {{donorName}}, we received {{amount}} on {{date}}...",
    "variables": ["donorName", "amount", "date"] } }
PUT /api/notification-templates/{templateKey} ADMIN

Update a template's subject and/or body. Changes are audit-logged.

Request body: 
{
  "subject": "New subject with {{variables}}",
  "body":    "Updated email body with {{variable}} substitution..."
}
📣 Campaigns — /api/campaigns
GET /api/campaigns/active?size=10 Public

List campaigns with status=ACTIVE. Used on the public donation page.

GET /api/campaigns?page=0&size=20 STAFF ADMIN

List all campaigns regardless of status (paginated).

GET /api/campaigns/{id} Any Auth

Get details of a single campaign.

POST /api/campaigns STAFF ADMIN

Create a new fundraising campaign.

Request body: 
{
  "campaignName":  "Winter Coats Drive",
  "description":   "Collect warm coats for families in need.",
  "goalAmount":    5000.00,
  "startDate":     "2025-11-01",
  "endDate":       "2025-12-31"
}
PUT /api/campaigns/{id} STAFF ADMIN

Update campaign details. Same body fields as POST.

PATCH /api/campaigns/{id}/close STAFF ADMIN

Mark a campaign as CLOSED. No new donations can be added to a closed campaign.

POST /api/campaigns/{id}/bulk-update STAFF ADMIN

Send a bulk email notification to all donors who have contributed to this campaign.

Request body: 
{
  "subject": "Campaign Update",
  "message": "Dear {{donorName}}, here is an update on {{campaignName}}..."
}

Email sent to all active donors with custom message and variable substitution (campaignName, donorName, etc).

DELETE /api/campaigns/{id} ADMIN

Delete a campaign. Fails with 409 if the campaign has any associated donations.

💰 Donations — /api/donations
POST /api/donations DONOR

Submit a new donation. The donor is taken from the JWT — no userId needed in the body.

Request body: 
{
  "campaignId":     3,
  "amount":         100.00,
  "paymentMethod":  "CREDIT_CARD",   // CREDIT_CARD | DEBIT_CARD | BANK_TRANSFER | CASH | CHEQUE
  "notes":          "Happy to help!" // optional
}
GET /api/donations/my?page=0&size=10&from=2025-01-01&to=2025-12-31&campaignId=3 DONOR

View own donation history. All query params are optional.

  • from / to — ISO date filter (YYYY-MM-DD)
  • campaignId — filter by a specific campaign
GET /api/donations/{id} STAFF CASE_MGR ADMIN

Fetch a single donation record by ID.

POST /api/donations/recurring DONOR

Schedule a recurring donation. The system will automatically process donations at the specified frequency.

Request body: 
{
  "campaignId":    3,
  "amount":        50.00,
  "frequency":     "MONTHLY",        // WEEKLY | MONTHLY
  "paymentMethod": "CREDIT_CARD",    // CREDIT_CARD | DEBIT_CARD | BANK_TRANSFER
  "notes":         "Monthly support" // optional
}
GET /api/donations/recurring/my DONOR

List all active recurring donations for the current donor.

DELETE /api/donations/recurring/{id} DONOR

Cancel a recurring donation schedule. Processing stops immediately.

📋 Aid Requests — /api/aid-requests
GET /api/aid-requests?page=0&size=20&status=PENDING STAFF CASE_MGR ADMIN

List aid requests with optional status filter.

Status values: PENDING · APPROVED · DENIED · FULFILLED

GET /api/aid-requests/{id} STAFF CASE_MGR ADMIN

Get a single aid request's full details.

POST /api/aid-requests STAFF ADMIN

Submit a new aid request on behalf of a beneficiary.

Request body: 
{
  "beneficiaryId":  12,
  "requestType":    "FOOD",           // FOOD | CLOTHING | MEDICAL | SHELTER | OTHER
  "description":    "Family of 4 needs emergency food supply",
  "urgencyLevel":   "HIGH",           // LOW | MEDIUM | HIGH | CRITICAL
  "inventoryItemId": 5,               // optional — link to inventory item
  "quantityNeeded": 2                 // optional
}
PATCH /api/aid-requests/{id}/approve CASE_MGR ADMIN

Approve a PENDING request. Status changes to APPROVED.

PATCH /api/aid-requests/{id}/deny?justification=Ineligible CASE_MGR ADMIN

Deny a PENDING request. justification query param (required) is stored as the denial reason.

PATCH /api/aid-requests/{id}/fulfill STAFF ADMIN

Mark an APPROVED request as FULFILLED. Decrements linked inventory item quantity if applicable.

📦 Inventory — /api/inventory
GET /api/inventory?page=0&size=20 STAFF ADMIN

List all inventory items (paginated).

GET /api/inventory/{id} STAFF ADMIN

Get a single inventory item by ID.

GET /api/inventory/low-stock STAFF ADMIN

Returns items whose current quantity is at or below their reorder threshold.

POST /api/inventory STAFF ADMIN

Add a new inventory item.

Request body: 
{
  "itemName":        "Winter Blanket",
  "category":        "CLOTHING",       // FOOD | CLOTHING | MEDICAL | SHELTER | OTHER
  "description":     "Heavy wool blanket",  // optional
  "quantityInStock": 50,
  "reorderThreshold": 10,              // alert when stock falls to this level
  "unit":            "pieces"          // optional, e.g. kg, litres, boxes
}
PUT /api/inventory/{id} STAFF ADMIN

Update an inventory item. Same body fields as POST.

DELETE /api/inventory/{id} ADMIN

Delete an inventory item. Fails if the item is linked to any unresolved aid requests.

📊 Reports — /api/reports
GET /api/reports/compliance?from=2025-01-01&to=2025-12-31&entityType=User&performedBy=admin@example.com LEADERSHIP ADMIN

Generate a compliance audit report from the audit trail. All query parameters are optional.

  • from / to — ISO date filter (YYYY-MM-DD) for date range
  • entityType — filter by entity (User, Campaign, Donation, AidRequest, etc.)
  • performedBy — filter by email of the action performer
Response includes: 
{ "status": "OK", "data": {
    "totalEvents": 150,
    "createCount": 40,
    "updateCount": 60,
    "deleteCount": 20,
    "securityEventCount": 30,
    "entries": [ {...audit log entries...} ] } }
⚠ Standard Error Response Format

All errors follow the same JSON envelope:

{
  "status":  "ERROR",
  "message": "Human-readable error description",
  "data":    null
}
HTTP CodeMeaning
400Validation error — check request body fields.
401Not authenticated — missing or expired JWT.
403Forbidden — your role does not permit this action, or account is locked/suspended.
404Resource not found.
409Conflict — e.g. duplicate email, campaign already has donations.
500Internal server error.
🖥 Quick cURL Examples

1. Create first admin (one-time setup):

curl -X POST http://localhost:8080/api/auth/setup \
  -H "Content-Type: application/json" \
  -d '{"fullName":"Admin","email":"admin@cams.org","password":"Admin@1234"}'

2. Login and capture token:

curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@cams.org","password":"Admin@1234"}'

3. Use token for a protected call (replace TOKEN):

curl http://localhost:8080/api/users \
  -H "Authorization: Bearer TOKEN"

4. Create a campaign:

curl -X POST http://localhost:8080/api/campaigns \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"campaignName":"Food Drive","description":"Monthly food drive","goalAmount":3000,"startDate":"2025-08-01","endDate":"2025-08-31"}'
